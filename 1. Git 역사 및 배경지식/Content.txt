# Git 역사 및 배경지식

1. 유닉스 -> 대형컴퓨터 운영체제
2. 대형컴퓨터 -> 리눅스 토발즈 -> 개인용 컴퓨터 OS -> 리눅스
3. GNU: General Public License -> 공개 소프트웨어 프로젝트
4. 리눅스는 GNU이고 GNU는 리눅스이다.
5. 소프트웨어는 누구에게나 공개하겠다.
6. 의미: A 소프트웨어가 GNU로 개발되었으면 A를 기반으로 B를 만들었다면 B도 공개해야 한다. (오픈 소스)
7. 리눅스 프로젝트를 만들려면 (a, b, c, d...) 여러 개발자가 필요하다.
8. 여러 개발자가 개발을 할 때는 문제가 생긴다. 사실상 지옥 형성
9. DVCS를 사용하여 버전을 관리했었다.
10. 근데 유료화했네?
11. 그래서 Git이 만들어졌다. 저런 기술적인 요소들을 접목하여 새로운 버전의 프로젝트를 만든 것이다.
12. 그래서 Git을 사용한다면 GPL을 따라야 한다. General Public License를 보면 무슨 소리인지 알겠지?
13. 다음에는 GitHub의 스토리를 알아보겠다.
